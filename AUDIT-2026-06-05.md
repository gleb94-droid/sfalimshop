# Sfalim Shop — Full-Site Pre-Launch Audit (Consolidated)

## Executive summary
The site has strong fundamentals across RTL, accessibility scaffolding, SEO, payment-integrity design, and performance, but several launch-blocking defects cluster around three themes: (1) the new flat-₪149 custom-shirt flow shipped with a keyboard/screen-reader-inaccessible toggle, white-on-orange contrast failures, and no purchase data path into the confirmation email; (2) the payment trust model rests on a client-writable `orders.total` and a stale/exploitable webhook in git that would reintroduce payment spoofing if redeployed; (3) launch-gate and structured-data drift — noindex is split across 3 files and the published JSON-LD still advertises pre-overhaul prices (₪89/119/95 vs ₪149). Analytics has zero e-commerce funnel events, so revenue and ad optimization would be blind at launch. Most issues are specific and quick to fix before going public.

## 🔴 Critical
- **[Payment]** `orders.total` is customer-controlled on INSERT — both server recompute and webhook match derive from a tamperable value (insert total:1 → charged ₪1, match passes) — `create-payment/index.ts:191-192; migration 20260531130000:22-28; App.jsx:6135` — server-validate/recompute total from the catalog + shipping table on INSERT and reject client-supplied total.
- **[Security/Payment]** Repo `tranzila-webhook` is the pre-hardening version (no query-back, no signature) — redeploying from git reintroduces payment spoofing via forged `{Response:'000', myid, sum}` — `tranzila-webhook/index.ts:70-97,141-205` — commit the deployed v12 (report.tranzila.com query-back, read order_group from u71) so git == prod.
- **[A11y]** Included-prints toggles are bare `<div onClick>` — unreachable by keyboard, silent to screen readers (WCAG 2.1.1 + 4.1.2) on a primary purchase control — `App.jsx:6685-6707` — make each a real `<button>`/role=switch with tabIndex, onKeyDown, aria-checked, and a labelled price.
- **[Analytics]** No `purchase`/`Purchase` event on successful payment return — GA4 has zero revenue, Meta can't attribute/optimize conversions — `App.jsx:3141-3146` — fire gtag purchase + fbq Purchase (transaction_id=order_group, value, currency 'ILS', items) with a once-per-order dedupe.
- **[Analytics]** No `begin_checkout`/`InitiateCheckout` at the Tranzila redirect — the last on-site touchpoint before off-site payment is dark — `App.jsx:7000-7041` — fire both events with cart items + pendingTotal right before redirect.
- **[Email]** New shirt details (model/size/colour/which prints) never reach the order-confirmation email — webhook selects only `product`, sends `"N items"` — `tranzila-webhook/index.ts:105,230-246` — also SELECT variant/color/print URLs/pet_name and pass a per-line items array.
- **[A11y]** White text on accent `#FF6B35` fails AA contrast (2.84:1) on Lock-position, step-2 Continue, and Same/Upload buttons — `App.jsx:6558,6718,6721,6757` — swap to `COLORS.accentBtn` `#C0501A` (4.77:1), already used correctly on sibling buttons.
- **[SEO/Legal]** Published `index.html` JSON-LD advertises stale prices (Tee ₪89, Oversize ₪119, Dri-FIT ₪95; priceRange ₪35-₪119) contradicting the flat ₪149 — wrong-price rich snippets, consumer-protection exposure — `index.html:108,147-151` — set all shirt offers to 149, priceRange to ₪35-₪149, add Lycra/Look/Stone-wash (reported by SEO + Legal + Overhaul).
- **[SEO]** noindex launch gate split across 3 files / 5 manual edits with no single source of truth — easy to half-launch (e.g. whole SPA stays deindexed or vice-versa) — `index.html:49-51; api/og.js:26; App.jsx:~1509` — drive all three from one env flag with a pre-launch assertion that they agree.
